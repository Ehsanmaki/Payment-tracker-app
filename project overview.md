# Project Overview

I want to build a personal Android application (Persian language UI) for tracking and analyzing my expenses and payment behavior.

Currently, I manage all of my expenses in an Excel spreadsheet. The goal of this application is to automate the process by detecting and parsing banking transaction SMS messages received on my phone, extracting payment information, and recording it in a structured format.

The application is for personal use only.

"Do not assume the Excel structure is the optimal database structure. Analyze it and propose a normalized database design."

Before implementation, please ask any questions necessary to improve the architecture, user experience, data model, and technical design.

I will provide:

1. Sample SMS messages from my banks.
2. My Excel template file.
3. Additional business rules if needed.
